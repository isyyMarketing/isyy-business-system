# Principii de lucru

Acest document definește principiile oficiale pentru ISYY Business System v1.0.

Principiile sunt folosite pentru a păstra claritatea, disciplina și direcția corectă în proiecte.

## Principii generale

- Nu se inventează certitudini.
- Dacă ceva este incert, se marchează ca incert.
- Fiecare idee se analizează critic.
- Soluțiile trebuie să fie modulare, clare, scalabile și ușor de întreținut.
- AI implementează cât mai mult, iar Isac coordonează și validează.
- Se lucrează etapizat, nu haotic.
- Nu se începe implementarea fără arhitectură și specificație clară.

## Reguli anti-haos

Pentru a evita lucrul dezorganizat:

- fiecare proiect începe cu o idee clară;
- ideea este analizată înainte de implementare;
- deciziile importante se documentează;
- taskurile mari se împart în pași mici;
- nu se schimbă direcția proiectului în mijlocul execuției fără oprire și confirmare;
- nu se amestecă analiza cu implementarea;
- nu se lucrează simultan la prea multe direcții fără prioritate.

## Reguli anti-halu

Pentru a evita concluziile false sau presupunerile necontrolate:

- AI nu trebuie să prezinte presupunerile ca adevăruri;
- informațiile incerte se marchează ca incerte;
- cifrele, estimările și promisiunile se tratează cu prudență;
- nu se garantează rezultate financiare;
- nu se iau decizii importante pe baza unei singure impresii;
- se cer clarificări când cerința este incompletă;
- se separă faptele de opinii.

## Reguli de decizie

O decizie bună trebuie să fie:

- clară;
- documentată;
- legată de un obiectiv;
- reversibilă când este posibil;
- justificată prin logică, nu doar prin entuziasm.

Înainte de o decizie importantă se verifică:

- ce problemă rezolvă;
- ce cost are;
- ce risc aduce;
- ce alternative există;
- ce se întâmplă dacă decizia este greșită.

## Reguli de scalare

Un proiect poate fi scalat doar dacă:

- rezolvă o problemă reală;
- are un client țintă clar;
- poate fi livrat repetabil;
- are costuri controlate;
- poate fi explicat simplu;
- are un model de venit;
- poate fi îmbunătățit pe baza feedbackului.

Nu se scalează haosul. Mai întâi se clarifică procesul, apoi se automatizează și se extinde.

## Reguli de colaborare cu AI

Colaborarea cu AI trebuie să fie clară și controlată.

ChatGPT este folosit pentru:

- analiză;
- arhitectură;
- mentorat;
- review;
- documentație;
- specificații.

Codex în VS Code este folosit pentru:

- implementare;
- modificarea fișierelor;
- dezvoltare de cod;
- completarea documentației;
- verificări locale.

Isac rămâne validatorul final.

AI poate accelera munca, dar nu înlocuiește decizia antreprenorială.
