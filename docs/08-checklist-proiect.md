# Checklist pentru proiect nou

Acest checklist se folosește înainte de începerea unui proiect nou în ISYY Business System v1.0.

Scopul este să clarifice ideea înainte de implementare.

## 1. Idee

- Care este ideea?
- Ce problemă rezolvă?
- De ce contează?
- Este o idee de serviciu, produs digital, automatizare sau agent AI?

## 2. Piață

- Există oameni sau firme care au această problemă?
- Problema este suficient de importantă?
- Există alternative deja pe piață?
- Ce ar face proiectul diferit sau mai simplu?

## 3. Client țintă

- Cine este clientul principal?
- Este persoană fizică sau firmă?
- Ce nivel de cunoștințe are?
- Ce buget ar putea avea?
- Unde poate fi găsit?

## 4. Model de venit

- Cum se poate câștiga bani?
- Este vânzare unică, abonament sau serviciu recurent?
- Prețul este realist?
- Există o cale clară spre minimum 2000 euro pe lună, fără garanție?

## 5. Costuri

- Ce costuri există la început?
- Ce costuri apar lunar?
- Sunt necesare unelte plătite?
- Este nevoie de hosting, domeniu, design sau reclame?

## 6. MVP

- Care este cea mai simplă versiune utilă?
- Ce funcționalități sunt obligatorii?
- Ce poate fi amânat?
- Cum se poate testa rapid?

## 7. Stack

- Ce tehnologii sunt necesare?
- Se folosește Node.js, Python sau ambele?
- Este nevoie de Docker?
- Este nevoie de design în Canva sau Figma?
- Este nevoie de integrare cu Gmail, Google Drive sau Google Calendar?

## 8. Riscuri

- Ce este incert?
- Ce poate eșua?
- Ce este prea complicat?
- Ce depinde de alte servicii?
- Ce trebuie validat înainte de investiție serioasă?

## 9. Lansare

- Unde se lansează?
- Cine vede prima versiune?
- Cum se colectează feedback?
- Ce înseamnă o lansare reușită?

## 10. Monetizare

- Când se cere plata?
- Cum se livrează valoarea?
- Stripe sau PayPal sunt necesare după lansare?
- Există o ofertă clară?

## 11. Scalare

- Ce se poate automatiza?
- Ce se poate transforma în produs digital?
- Ce poate fi repetat pentru mai mulți clienți?
- Ce poate deveni agent AI?
- Ce trebuie documentat pentru a crește mai ușor?

## Decizie finală

Înainte de implementare, proiectul trebuie încadrat într-una dintre variante:

- continuă spre arhitectură și specificație;
- rămâne în analiză;
- se amână;
- se abandonează.

Nu se începe implementarea dacă răspunsurile principale sunt neclare.
