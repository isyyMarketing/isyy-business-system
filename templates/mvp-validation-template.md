# Template MVP si validare

Acest template se foloseste inainte de construirea sau lansarea unui MVP.

## Ce problema rezolva MVP-ul

Descrie problema principala pe care MVP-ul trebuie sa o rezolve.

## Functii minime

MVP-ul include doar functiile necesare pentru testare:

- functie 1;
- functie 2;
- functie 3.

## Ce nu intra in MVP

Se amana:

- functie avansata 1;
- automatizare complexa;
- design final;
- integrare care nu este necesara pentru test.

## Criterii pentru validare

MVP-ul este validat daca:

- utilizatorul intelege ce face produsul;
- problema principala este rezolvata;
- exista interes real;
- exista feedback concret;
- exista posibilitate clara de monetizare.

## Criterii pentru lansare

MVP-ul poate fi lansat daca:

- functioneaza stabil pentru scopul minim;
- mesajul pentru client este clar;
- exista metoda de contact sau cumparare;
- riscurile cunoscute sunt documentate;
- Isac aproba lansarea.

## Criterii pentru oprire sau schimbare strategie

Se opreste sau se schimba strategia daca:

- problema nu este importanta pentru client;
- clientul nu intelege valoarea;
- costul de livrare este prea mare;
- nu exista interes dupa testare;
- apar riscuri mai mari decat beneficiile.
